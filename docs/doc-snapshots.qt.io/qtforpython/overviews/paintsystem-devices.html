Paint Devices and Backends¶

Creating a Paint Device¶

The PySide2.QtGui.QPaintDevice class is the base class of objects that can be painted, i.e. PySide2.QtGui.QPainter can draw on any PySide2.QtGui.QPaintDevice subclass. PySide2.QtGui.QPaintDevice ‘s drawing capabilities are among others implemented by PySide2.QtWidgets.QWidget , PySide2.QtGui.QImage , PySide2.QtGui.QPixmap , PySide2.QtGui.QPicture , PySide2.QtPrintSupport.QPrinter , and QOpenGLPaintDevice .

Widget

The PySide2.QtWidgets.QWidget class is the base class of user interface elements in the Qt Widgets module. It receives mouse, keyboard and other events from the window system, and paints a representation of itself on the screen.

Image

The PySide2.QtGui.QImage class provides a hardware-independent image representation which is designed and optimized for I/O, and for direct pixel access and manipulation. PySide2.QtGui.QImage supports several image formats including monochrome, 8-bit, 32-bit and alpha-blended images.

One advantage of using PySide2.QtGui.QImage as a paint device is that it is possible to guarantee the pixel exactness of any drawing operation in a platform-independent way. Another benefit is that the painting can be performed in another thread than the current GUI thread.

Pixmap

The PySide2.QtGui.QPixmap class is an off-screen image representation which is designed and optimized for showing images on screen. Unlike PySide2.QtGui.QImage , the pixel data in a pixmap is internal and is managed by the underlying window system, i.e. pixels can only be accessed through PySide2.QtGui.QPainter functions or by converting the PySide2.QtGui.QPixmap to a PySide2.QtGui.QImage .

To optimize drawing with PySide2.QtGui.QPixmap , Qt provides the PySide2.QtGui.QPixmapCache class which can be used to store temporary pixmaps that are expensive to generate without using more storage space than the cache limit.

Qt also provides the PySide2.QtGui.QBitmap convenience class, inheriting PySide2.QtGui.QPixmap . PySide2.QtGui.QBitmap guarantees monochrome (1-bit depth) pixmaps, and is mainly used for creating custom PySide2.QtGui.QCursor and PySide2.QtGui.QBrush objects, constructing PySide2.QtGui.QRegion objects.

OpenGL Paint Device

As mentioned previously, Qt is offering classes that makes it easy to use OpenGL in Qt applications. For example, the QOpenGLPaintDevice enables the OpenGL API for rendering with PySide2.QtGui.QPainter .

Picture

The PySide2.QtGui.QPicture class is a paint device that records and replays PySide2.QtGui.QPainter commands. A picture serializes painter commands to an IO device in a platform-independent format. PySide2.QtGui.QPicture is also resolution independent, i.e. a PySide2.QtGui.QPicture can be displayed on different devices (for example svg, pdf, ps, printer and screen) looking the same.

Qt provides the QPicture.load() and QPicture.save() functions as well as streaming operators for loading and saving pictures.

Custom Backends

Support for a new backend can be implemented by deriving from the PySide2.QtGui.QPaintDevice class and reimplementing the virtual QPaintDevice.paintEngine() function to tell PySide2.QtGui.QPainter which paint engine should be used to draw on this particular device. To actually be able to draw on the device, this paint engine must be a custom paint engine created by deriving from the PySide2.QtGui.QPaintEngine class.
