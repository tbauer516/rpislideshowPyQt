Working with the DOM Tree¶

DOM Level 2 is a W3C Recommendation for XML interfaces that maps the constituents of an XML document to a tree structure. The specification of DOM Level 2 can be found at http://www.w3.org/DOM/.

Introduction to DOM¶

DOM provides an interface to access and change the content and structure of an XML file. It makes a hierarchical view of the document (a tree view). Thus – in contrast to the SAX2 interface – an object model of the document is resident in memory after parsing which makes manipulation easy.

All DOM nodes in the document tree are subclasses of PySide2.QtXml.QDomNode . The document itself is represented as a PySide2.QtXml.QDomDocument object.

Here are the available node classes and their potential child classes:

  • PySide2.QtXml.QDomNotation : No children

With PySide2.QtXml.QDomNodeList and PySide2.QtXml.QDomNamedNodeMap two collection classes are provided: PySide2.QtXml.QDomNodeList is a list of nodes, and PySide2.QtXml.QDomNamedNodeMap is used to handle unordered sets of nodes (often used for attributes).

The PySide2.QtXml.QDomImplementation class allows the user to query features of the DOM implementation.

To get started please refer to the PySide2.QtXml.QDomDocument documentation. You might also want to take a look at the DOM Bookmarks example , which illustrates how to read and write an XML bookmark file (XBEL) using DOM.
