SQL Programming¶

Database integration for Qt applications

Database integration for Qt applications.

This overview assumes that you have at least a basic knowledge of SQL. You should be able to understand simple SELECT , INSERT , UPDATE , and DELETE statements. Although the PySide2.QtSql.QSqlTableModel class provides an interface to database browsing and editing that does not require a knowledge of SQL, a basic understanding of SQL is highly recommended. A standard text covering SQL databases is An Introduction to Database Systems (7th Ed.) by C. J. Date, ISBN 0201385902.

Database Classes¶

These classes provide access to SQL databases.

The SQL classes are divided into three layers:

Driver Layer¶

This comprises the classes PySide2.QtSql.QSqlDriver , QSqlDriverCreator , PySide2.QtSql.QSqlDriverCreatorBase , QSqlDriverPlugin , and PySide2.QtSql.QSqlResult .

This layer provides the low-level bridge between the specific databases and the SQL API layer. See SQL Database Drivers for more information.

SQL API Layer¶

These classes provide access to databases. Connections are made using the PySide2.QtSql.QSqlDatabase class. Database interaction is achieved by using the PySide2.QtSql.QSqlQuery class. In addition to PySide2.QtSql.QSqlDatabase and PySide2.QtSql.QSqlQuery , the SQL API layer is supported by PySide2.QtSql.QSqlError , PySide2.QtSql.QSqlField , PySide2.QtSql.QSqlIndex , and PySide2.QtSql.QSqlRecord .

User Interface Layer¶

These classes link the data from a database to data-aware widgets. They include PySide2.QtSql.QSqlQueryModel , PySide2.QtSql.QSqlTableModel , and PySide2.QtSql.QSqlRelationalTableModel . These classes are designed to work with Qt’s model/view framework .

Note that a PySide2.QtCore.QCoreApplication object must be instantiated before using any of these classes.
