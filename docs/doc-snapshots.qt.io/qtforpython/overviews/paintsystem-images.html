Reading and Writing Image Files¶

The most common way to read images is through PySide2.QtGui.QImage and PySide2.QtGui.QPixmap ‘s constructors, or by calling the QImage.load() and QPixmap.load() functions. In addition, Qt provides the PySide2.QtGui.QImageReader class which gives more control over the process. Depending on the underlying support in the image format, the functions provided by the class can save memory and speed up loading of images.

Likewise, Qt provides the PySide2.QtGui.QImageWriter class which supports setting format specific options, such as the gamma level, compression level and quality, prior to storing the image. If you do not need such options, you can use QImage.save() or QPixmap.save() instead.

The PySide2.QtGui.QImageReader and PySide2.QtGui.QImageWriter classes rely on the PySide2.QtGui.QImageIOHandler class which is the common image I/O interface for all image formats in Qt. PySide2.QtGui.QImageIOHandler objects are used internally by PySide2.QtGui.QImageReader and PySide2.QtGui.QImageWriter to add support for different image formats to Qt.

A list of the supported file formats are available through the QImageReader.supportedImageFormats() and QImageWriter.supportedImageFormats() functions. Qt supports several file formats by default, and in addition new formats can be added as plugins. The currently supported formats are listed in the PySide2.QtGui.QImageReader and PySide2.QtGui.QImageWriter class documentation.

Qt’s plugin mechanism can also be used to write a custom image format handler. This is done by deriving from the PySide2.QtGui.QImageIOHandler class, and creating a QImageIOPlugin object which is a factory for creating PySide2.QtGui.QImageIOHandler objects. When the plugin is installed, PySide2.QtGui.QImageReader and PySide2.QtGui.QImageWriter will automatically load the plugin and start using it.
