Drop Site Example¶

Example shows how to distinguish the various MIME formats available in a drag and drop operation

The example shows how to distinguish the various MIME formats available in a drag and drop operation.

The Drop Site example accepts drops from other applications, and displays the MIME formats provided by the drag object.

There are two classes, DropArea and DropSiteWindow , and a main() function in this example. A DropArea object is instantiated in DropSiteWindow ; a DropSiteWindow object is then invoked in the main() function.

DropArea Class Definition¶

The DropArea class is a subclass of PySide2.QtWidgets.QLabel with a public slot, clear() , and a changed() signal.

class DropArea : public QLabel
{
    Q_OBJECT

public:
    explicit DropArea(QWidget *parent = nullptr);

public slots:
    void clear();

signals:
    void changed(const QMimeData *mimeData = nullptr);

In addition, DropArea also contains a private instance of PySide2.QtWidgets.QLabel and reimplementations of four PySide2.QtWidgets.QWidget event handlers:

  1. QWidget.dragEnterEvent()
  2. QWidget.dragMoveEvent()
  3. QWidget.dragLeaveEvent()
  4. QWidget.dropEvent()

These event handlers are further explained in the implementation of the DropArea class.

protected:
    void dragEnterEvent(QDragEnterEvent *event) override;
    void dragMoveEvent(QDragMoveEvent *event) override;
    void dragLeaveEvent(QDragLeaveEvent *event) override;
    void dropEvent(QDropEvent *event) override;

private:
    QLabel *label;
};

DropArea Class Implementation¶

In the DropArea constructor, we set the minimum size to 200x200 pixels, the frame style to both QFrame.Sunken and QFrame.StyledPanel , and we align its contents to the center.

DropArea::DropArea(QWidget *parent)
    : QLabel(parent)
{
    setMinimumSize(200, 200);
    setFrameStyle(QFrame::Sunken | QFrame::StyledPanel);
    setAlignment(Qt::AlignCenter);
    setAcceptDrops(true);
    setAutoFillBackground(true);
    clear();
}

Also, we enable drop events in DropArea by setting the QWidget.acceptDrops() property to true . Then, we enable the QWidget.autoFillBackground() property and invoke the clear() function.

The QWidget.dragEnterEvent() event handler is called when a drag is in progress and the mouse enters the DropArea object. For the DropSite example, when the mouse enters DropArea , we set its text to “<drop content>” and highlight its background.

The mimeData object can contain one of the following objects: an image, HTML text, plain text, or a list of URLs.

  • If mimeData contains an image, we display it in DropArea with QLabel.setPixmap() .
  • If mimeData contains HTML, we display it with QLabel.setText() and set DropArea’s text format as Qt.RichText .
  • If mimeData contains plain text, we display it with QLabel.setText() and set DropArea’s text format as Qt.PlainText . In the event that mimeData contains URLs, we iterate through the list of URLs to display them on individual lines.
  • If mimeData contains other types of objects, we set DropArea’s text, with QLabel.setText() to “Cannot display data” to inform the user.

We then set DropArea ‘s QWidget.backgroundRole() to QPalette.Dark and we accept event ‘s proposed action.

DropSiteWindow Class Definition¶

The DropSiteWindow class contains a constructor and a public slot, updateFormatsTable() .

class DropSiteWindow : public QWidget
{
    Q_OBJECT

public:
    DropSiteWindow();

public slots:
    void updateFormatsTable(const QMimeData *mimeData);

private:
    DropArea *dropArea;
    QLabel *abstractLabel;
    QTableWidget *formatsTable;

    QPushButton *clearButton;
    QPushButton *quitButton;
    QDialogButtonBox *buttonBox;
};

The class also contains a private instance of DropArea , dropArea , PySide2.QtWidgets.QLabel , abstractLabel , PySide2.QtWidgets.QTableWidget , formatsTable , PySide2.QtWidgets.QDialogButtonBox , buttonBox , and two PySide2.QtWidgets.QPushButton objects, clearButton and quitButton .

DropSiteWindow Class Implementation¶

In the constructor of DropSiteWindow , we instantiate abstractLabel and set its wordWrap property to true . We also call the QLabel.adjustSize() function to adjust abstractLabel ‘s size according to its contents.

DropSiteWindow::DropSiteWindow()
{
    abstractLabel = new QLabel(tr("This example accepts drags from other "
                                  "applications and displays the MIME types "
                                  "provided by the drag object."));
    abstractLabel->setWordWrap(true);
    abstractLabel->adjustSize();

Then we instantiate dropArea and connect its changed() signal to DropSiteWindow ‘s updateFormatsTable() slot.

dropArea = new DropArea;
connect(dropArea, &DropArea::changed,
        this, &DropSiteWindow::updateFormatsTable);

We now set up the PySide2.QtWidgets.QTableWidget object, formatsTable . Its horizontal header is set using a PySide2.QtCore.QStringList object, labels . The number of columms are set to two and the table is not editable. Also, the formatTable ‘s horizontal header is formatted to ensure that its second column stretches to occupy additional space available.

QStringList labels;
labels << tr("Format") << tr("Content");

formatsTable = new QTableWidget;
formatsTable->setColumnCount(2);
formatsTable->setEditTriggers(QAbstractItemView::NoEditTriggers);
formatsTable->setHorizontalHeaderLabels(labels);
formatsTable->horizontalHeader()->setStretchLastSection(true);

Two PySide2.QtWidgets.QPushButton objects, clearButton and quitButton , are instantiated and added to buttonBox - a PySide2.QtWidgets.QDialogButtonBox object. We use PySide2.QtWidgets.QDialogButtonBox here to ensure that the push buttons are presented in a layout that conforms to the platform’s style.

clearButton = new QPushButton(tr("Clear"));
quitButton = new QPushButton(tr("Quit"));

buttonBox = new QDialogButtonBox;
buttonBox->addButton(clearButton, QDialogButtonBox::ActionRole);
buttonBox->addButton(quitButton, QDialogButtonBox::RejectRole);

connect(quitButton, &QAbstractButton::clicked, this, &QWidget::close);
connect(clearButton, &QAbstractButton::clicked, dropArea, &DropArea::clear);

The QPushButton.clicked() signals for quitButton and clearButton are connected to QWidget.close() and clear() , respectively.

For the layout, we use a PySide2.QtWidgets.QVBoxLayout , mainLayout , to arrange our widgets vertically. We also set the window title to “Drop Site” and the minimum size to 350x500 pixels.

    QVBoxLayout *mainLayout = new QVBoxLayout(this);
    mainLayout->addWidget(abstractLabel);
    mainLayout->addWidget(dropArea);
    mainLayout->addWidget(formatsTable);
    mainLayout->addWidget(buttonBox);

    setWindowTitle(tr("Drop Site"));
    setMinimumSize(350, 500);
}

We move on to the updateFormatsTable() function. This function updates the formatsTable , displaying the MIME formats of the object dropped onto the DropArea object. First, we set PySide2.QtWidgets.QTableWidget ‘s rowCount property to 0. Then, we validate to ensure that the PySide2.QtCore.QMimeData object passed in is a valid object.

<Code snippet "draganddrop/dropsite/dropsitewindow.cpp:updateFormatsTable() part1" not found>

Once we are sure that mimeData is valid, we iterate through its supported formats using the foreach keyword . This keyword has the following format:

foreach(variable, container)

In our example, format is the variable and the container is a PySide2.QtCore.QStringList , obtained from mimeData->formats() .

Note

The QMimeData.formats() function returns a PySide2.QtCore.QStringList object, containing all the formats supported by the mimeData .

<Code snippet "draganddrop/dropsite/dropsitewindow.cpp:updateFormatsTable() part2" not found>

Within each iteration, we create a PySide2.QtWidgets.QTableWidgetItem , formatItem and we set its flags to Qt.ItemIsEnabled , and its text alignment to Qt.AlignTop and Qt.AlignLeft .

A PySide2.QtCore.QString object, text , is customized to display data according to the contents of format . We invoke PySide2.QtCore.QString ‘s QString.simplified() function on text , to obtain a string that has no additional space before, after or in between words.

<Code snippet "draganddrop/dropsite/dropsitewindow.cpp:updateFormatsTable() part3" not found>

If format contains a list of URLs, we iterate through them, using spaces to separate them. On the other hand, if format contains an image, we display the data by converting the text to hexadecimal.

<Code snippet "draganddrop/dropsite/dropsitewindow.cpp:updateFormatsTable() part4" not found>

Once text has been customized to contain the appropriate data, we insert both format and text into formatsTable with QTableWidget.setItem() . Lastly, we invoke QTableView.resizeColumnToContents() on formatsTable ‘s first column.
