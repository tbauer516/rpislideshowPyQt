Styling¶

Qt’s built-in widgets use the PySide2.QtWidgets.QStyle class to perform nearly all of their drawing. PySide2.QtWidgets.QStyle is an abstract base class that encapsulates the look and feel of a GUI, and can be used to make the widgets look exactly like the equivalent native widgets or to give the widgets a custom look.

Qt provides a set of PySide2.QtWidgets.QStyle subclasses that emulate the native look of the different platforms supported by Qt (QWindowsStyle, QMacStyle, etc.). These styles are built into the Qt GUI module, other styles can be made available using Qt’s plugin mechansim.

Most functions for drawing style elements take four arguments:

  • an enum value specifying which graphical element to draw
  • a PySide2.QtWidgets.QStyleOption object specifying how and where to render that element
  • a PySide2.QtGui.QPainter object that should be used to draw the element
  • a PySide2.QtWidgets.QWidget object on which the drawing is performed (optional)

The style gets all the information it needs to render the graphical element from the PySide2.QtWidgets.QStyleOption class. The widget is passed as the last argument in case the style needs it to perform special effects (such as animated default buttons on macOS ), but it isn’t mandatory. In fact, PySide2.QtWidgets.QStyle can be used to draw on any paint device (not just widgets), in which case the widget argument is a zero pointer.

The paint system also provides the PySide2.QtWidgets.QStylePainter class inheriting from PySide2.QtGui.QPainter . PySide2.QtWidgets.QStylePainter is a convenience class for drawing PySide2.QtWidgets.QStyle elements inside a widget, and extends PySide2.QtGui.QPainter with a set of high-level drawing functions implemented on top of PySide2.QtWidgets.QStyle ‘s API. The advantage of using PySide2.QtWidgets.QStylePainter is that the parameter lists get considerably shorter.
