Detailed Description¶

The PySide2.QtWidgets.QHeaderView class provides a header row or header column for item views.

A PySide2.QtWidgets.QHeaderView displays the headers used in item views such as the PySide2.QtWidgets.QTableView and PySide2.QtWidgets.QTreeView classes. It takes the place of Qt3’s QHeader class previously used for the same purpose, but uses the Qt’s model/view architecture for consistency with the item view classes.

The PySide2.QtWidgets.QHeaderView class is one of the Model/View Classes and is part of Qt’s model/view framework .

The header gets the data for each section from the model using the QAbstractItemModel.headerData() function. You can set the data by using QAbstractItemModel.setHeaderData() .

Each header has an PySide2.QtWidgets.QHeaderView.orientation() and a number of sections, given by the PySide2.QtWidgets.QHeaderView.count() function. A section refers to a part of the header - either a row or a column, depending on the orientation.

Sections can be moved and resized using PySide2.QtWidgets.QHeaderView.moveSection() and PySide2.QtWidgets.QHeaderView.resizeSection() ; they can also be hidden and shown with PySide2.QtWidgets.QHeaderView.hideSection() and PySide2.QtWidgets.QHeaderView.showSection() .

Each section of a header is described by a section ID, specified by its section(), and can be located at a particular PySide2.QtWidgets.QHeaderView.visualIndex() in the header. A section can have a sort indicator set with PySide2.QtWidgets.QHeaderView.setSortIndicator() ; this indicates whether the items in the associated item view will be sorted in the order given by the section.

For a horizontal header the section is equivalent to a column in the model, and for a vertical header the section is equivalent to a row in the model.

Moving Header Sections¶

A header can be fixed in place, or made movable with PySide2.QtWidgets.QHeaderView.setSectionsMovable() . It can be made clickable with PySide2.QtWidgets.QHeaderView.setSectionsClickable() , and has resizing behavior in accordance with PySide2.QtWidgets.QHeaderView.setSectionResizeMode() .

Note

Double-clicking on a header to resize a section only applies for visible rows.

A header will emit PySide2.QtWidgets.QHeaderView.sectionMoved() if the user moves a section, PySide2.QtWidgets.QHeaderView.sectionResized() if the user resizes a section, and PySide2.QtWidgets.QHeaderView.sectionClicked() as well as PySide2.QtWidgets.QHeaderView.sectionHandleDoubleClicked() in response to mouse clicks. A header will also emit PySide2.QtWidgets.QHeaderView.sectionCountChanged() .

You can identify a section using the PySide2.QtWidgets.QHeaderView.logicalIndex() and PySide2.QtWidgets.QHeaderView.logicalIndexAt() functions, or by its index position, using the PySide2.QtWidgets.QHeaderView.visualIndex() and PySide2.QtWidgets.QHeaderView.visualIndexAt() functions. The visual index will change if a section is moved, but the logical index will not change.

Appearance¶

PySide2.QtWidgets.QTableWidget and PySide2.QtWidgets.QTableView create default headers. If you want the headers to be visible, you can use PySide2.QtWidgets.QFrame.setVisible() .

Not all Qt.ItemDataRole s will have an effect on a PySide2.QtWidgets.QHeaderView . If you need to draw other roles, you can subclass PySide2.QtWidgets.QHeaderView and reimplement PySide2.QtWidgets.QHeaderView.paintEvent() . PySide2.QtWidgets.QHeaderView respects the following item data roles: TextAlignmentRole , DisplayRole , FontRole , DecorationRole , ForegroundRole , and BackgroundRole .

Note

Each header renders the data for each section itself, and does not rely on a delegate. As a result, calling a header’s PySide2.QtWidgets.QAbstractItemView.setItemDelegate() function will have no effect.

PySide2.QtWidgets.QHeaderView.ResizeMode¶

The resize mode specifies the behavior of the header sections. It can be set on the entire header view or on individual sections using PySide2.QtWidgets.QHeaderView.setSectionResizeMode() .

Constant Description
QHeaderView.Interactive The user can resize the section. The section can also be resized programmatically using PySide2.QtWidgets.QHeaderView.resizeSection() . The section size defaults to PySide2.QtWidgets.QHeaderView.defaultSectionSize() . (See also PySide2.QtWidgets.QHeaderView.cascadingSectionResizes() .)
QHeaderView.Fixed The user cannot resize the section. The section can only be resized programmatically using PySide2.QtWidgets.QHeaderView.resizeSection() . The section size defaults to PySide2.QtWidgets.QHeaderView.defaultSectionSize() .
QHeaderView.Stretch PySide2.QtWidgets.QHeaderView will automatically resize the section to fill the available space. The size cannot be changed by the user or programmatically.
QHeaderView.ResizeToContents PySide2.QtWidgets.QHeaderView will automatically resize the section to its optimal size based on the contents of the entire column or row. The size cannot be changed by the user or programmatically. (This value was introduced in 4.2)

The following values are obsolete:

Constant Description
QHeaderView.Custom Use Fixed instead.

PySide2.QtWidgets.QHeaderView.setResizeContentsPrecision(precision)¶
Parameters:precision – PySide2.QtCore.int

Sets how precise PySide2.QtWidgets.QHeaderView should calculate the size when ResizeToContents is used. A low value will provide a less accurate but fast auto resize while a higher value will provide a more accurate resize that however can be slow.

The number precision specifies how many sections that should be consider when calculating the preferred size.

The default value is 1000 meaning that a horizontal column with auto-resize will look at maximum 1000 rows on calculating when doing an auto resize.

Special value 0 means that it will look at only the visible area. Special value -1 will imply looking at all elements.

This value is used in QTableView.sizeHintForColumn() , QTableView.sizeHintForRow() and QTreeView.sizeHintForColumn() . Reimplementing these functions can make this function not having an effect.

PySide2.QtWidgets.QHeaderView.setSortIndicator(logicalIndex, order)¶
Parameters:
  • logicalIndex – PySide2.QtCore.int
  • order – PySide2.QtCore.Qt.SortOrder

Sets the sort indicator for the section specified by the given logicalIndex in the direction specified by order , and removes the sort indicator from any other section that was showing it.

logicalIndex may be -1, in which case no sort indicator will be shown and the model will return to its natural, unsorted order. Note that not all models support this and may even crash in this case.
