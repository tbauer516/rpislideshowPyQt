Detailed Description¶

The PySide2.QtWidgets.QCompleter class provides completions based on an item model.

You can use PySide2.QtWidgets.QCompleter to provide auto completions in any Qt widget, such as PySide2.QtWidgets.QLineEdit and PySide2.QtWidgets.QComboBox . When the user starts typing a word, PySide2.QtWidgets.QCompleter suggests possible ways of completing the word, based on a word list. The word list is provided as a PySide2.QtCore.QAbstractItemModel . (For simple applications, where the word list is static, you can pass a PySide2.QtCore.QStringList to PySide2.QtWidgets.QCompleter ‘s constructor.)

Basic Usage¶

A PySide2.QtWidgets.QCompleter is used typically with a PySide2.QtWidgets.QLineEdit or PySide2.QtWidgets.QComboBox . For example, here’s how to provide auto completions from a simple word list in a PySide2.QtWidgets.QLineEdit :

wordList = ["alpha", "omega", "omicron", "zeta"]

lineEdit = QLineEdit(self)

completer = QCompleter(wordList, self)
completer.setCaseSensitivity(Qt.CaseInsensitive)
lineEdit.setCompleter(completer)

A PySide2.QtWidgets.QFileSystemModel can be used to provide auto completion of file names. For example:

completer = QCompleter(self)
completer.setModel(QDirModel(completer))
lineEdit.setCompleter(completer)

To set the model on which PySide2.QtWidgets.QCompleter should operate, call PySide2.QtWidgets.QCompleter.setModel() . By default, PySide2.QtWidgets.QCompleter will attempt to match the completion prefix (i.e., the word that the user has started typing) against the Qt.EditRole data stored in column 0 in the model case sensitively. This can be changed using PySide2.QtWidgets.QCompleter.setCompletionRole() , PySide2.QtWidgets.QCompleter.setCompletionColumn() , and PySide2.QtWidgets.QCompleter.setCaseSensitivity() .

If the model is sorted on the column and role that are used for completion, you can call PySide2.QtWidgets.QCompleter.setModelSorting() with either QCompleter.CaseSensitivelySortedModel or QCompleter.CaseInsensitivelySortedModel as the argument. On large models, this can lead to significant performance improvements, because PySide2.QtWidgets.QCompleter can then use binary search instead of linear search. The binary search only works when the PySide2.QtWidgets.QCompleter.filterMode() is Qt.MatchStartsWith .

The model can be a list model , a table model , or a tree model . Completion on tree models is slightly more involved and is covered in the Handling Tree Models section below.

The PySide2.QtWidgets.QCompleter.completionMode() determines the mode used to provide completions to the user.

Iterating Through Completions¶

To retrieve a single candidate string, call PySide2.QtWidgets.QCompleter.setCompletionPrefix() with the text that needs to be completed and call PySide2.QtWidgets.QCompleter.currentCompletion() . You can iterate through the list of completions as below:

i = 0
while completer.setCurrentRow(i):
    print "%s is match number %d" % (completer.currentCompletion(), i)
    i += 1

PySide2.QtWidgets.QCompleter.completionCount() returns the total number of completions for the current prefix. PySide2.QtWidgets.QCompleter.completionCount() should be avoided when possible, since it requires a scan of the entire model.

The Completion Model¶

PySide2.QtWidgets.QCompleter.completionModel() return a list model that contains all possible completions for the current completion prefix, in the order in which they appear in the model. This model can be used to display the current completions in a custom view. Calling PySide2.QtWidgets.QCompleter.setCompletionPrefix() automatically refreshes the completion model.

Handling Tree Models¶

PySide2.QtWidgets.QCompleter can look for completions in tree models, assuming that any item (or sub-item or sub-sub-item) can be unambiguously represented as a string by specifying the path to the item. The completion is then performed one level at a time.

Let’s take the example of a user typing in a file system path. The model is a (hierarchical) PySide2.QtWidgets.QFileSystemModel . The completion occurs for every element in the path. For example, if the current text is C:\Wind , PySide2.QtWidgets.QCompleter might suggest Windows to complete the current path element. Similarly, if the current text is C:\Windows\Sy , PySide2.QtWidgets.QCompleter might suggest System .

For this kind of completion to work, PySide2.QtWidgets.QCompleter needs to be able to split the path into a list of strings that are matched at each level. For C:\Windows\Sy , it needs to be split as “C:”, “Windows” and “Sy”. The default implementation of PySide2.QtWidgets.QCompleter.splitPath() , splits the PySide2.QtWidgets.QCompleter.completionPrefix() using QDir.separator() if the model is a PySide2.QtWidgets.QFileSystemModel .

To provide completions, PySide2.QtWidgets.QCompleter needs to know the path from an index. This is provided by PySide2.QtWidgets.QCompleter.pathFromIndex() . The default implementation of PySide2.QtWidgets.QCompleter.pathFromIndex() , returns the data for the edit role for list models and the absolute file path if the mode is a PySide2.QtWidgets.QFileSystemModel .

PySide2.QtWidgets.QCompleter.CompletionMode¶

This enum specifies how completions are provided to the user.

Constant Description
QCompleter.PopupCompletion Current completions are displayed in a popup window.
QCompleter.InlineCompletion Completions appear inline (as selected text).
QCompleter.UnfilteredPopupCompletion All possible completions are displayed in a popup window with the most likely suggestion indicated as current.

PySide2.QtWidgets.QCompleter.complete([rect=QRect()])¶
Parameters:rect – PySide2.QtCore.QRect

For QCompleter.PopupCompletion and QCompletion::UnfilteredPopupCompletion modes, calling this function displays the popup displaying the current completions. By default, if rect is not specified, the popup is displayed on the bottom of the PySide2.QtWidgets.QCompleter.widget() . If rect is specified the popup is displayed on the left edge of the rectangle.

For QCompleter.InlineCompletion mode, the PySide2.QtWidgets.QCompleter.highlighted() signal is fired with the current completion.

PySide2.QtWidgets.QCompleter.completionModel()¶
Return type:PySide2.QtCore.QAbstractItemModel

Returns the completion model. The completion model is a read-only list model that contains all the possible matches for the current completion prefix. The completion model is auto-updated to reflect the current completions.

Note

The return value of this function is defined to be an PySide2.QtCore.QAbstractItemModel purely for generality. This actual kind of model returned is an instance of an PySide2.QtCore.QAbstractProxyModel subclass.

PySide2.QtWidgets.QCompleter.setModel(c)¶
Parameters:c – PySide2.QtCore.QAbstractItemModel

Sets the model which provides completions to model . The model can be list model or a tree model. If a model has been already previously set and it has the PySide2.QtWidgets.QCompleter as its parent, it is deleted.

For convenience, if model is a PySide2.QtWidgets.QFileSystemModel , PySide2.QtWidgets.QCompleter switches its PySide2.QtWidgets.QCompleter.caseSensitivity() to Qt.CaseInsensitive on Windows and Qt.CaseSensitive on other platforms.

PySide2.QtWidgets.QCompleter.setPopup(popup)¶
Parameters:popup – PySide2.QtWidgets.QAbstractItemView

Sets the popup used to display completions to popup . PySide2.QtWidgets.QCompleter takes ownership of the view.

A PySide2.QtWidgets.QListView is automatically created when the PySide2.QtWidgets.QCompleter.completionMode() is set to QCompleter.PopupCompletion or QCompleter.UnfilteredPopupCompletion . The default popup displays the PySide2.QtWidgets.QCompleter.completionColumn() .

Ensure that this function is called before the view settings are modified. This is required since view’s properties may require that a model has been set on the view (for example, hiding columns in the view requires a model to be set on the view).

PySide2.QtWidgets.QCompleter.splitPath(path)¶
Parameters:path – unicode
Return type:list of strings

Splits the given path into strings that are used to match at each level in the PySide2.QtWidgets.QCompleter.model() .

The default implementation of splits a file system path based on QDir.separator() when the sourceModel() is a PySide2.QtWidgets.QFileSystemModel .

When used with list models, the first item in the returned list is used for matching.
