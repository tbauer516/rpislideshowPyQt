Detailed Description¶

The PySide2.QtXml.QDomElement class represents one element in the DOM tree.

Elements have a PySide2.QtXml.QDomElement.tagName() and zero or more attributes associated with them. The tag name can be changed with PySide2.QtXml.QDomElement.setTagName() .

Element attributes are represented by PySide2.QtXml.QDomAttr objects that can be queried using the PySide2.QtXml.QDomElement.attribute() and PySide2.QtXml.QDomElement.attributeNode() functions. You can set attributes with the PySide2.QtXml.QDomElement.setAttribute() and PySide2.QtXml.QDomElement.setAttributeNode() functions. Attributes can be removed with PySide2.QtXml.QDomElement.removeAttribute() . There are namespace-aware equivalents to these functions, i.e. PySide2.QtXml.QDomElement.setAttributeNS() , PySide2.QtXml.QDomElement.setAttributeNodeNS() and PySide2.QtXml.QDomElement.removeAttributeNS() .

If you want to access the text of a node use PySide2.QtXml.QDomElement.text() , e.g.

e = # some QDomElement...
#...
s = e.text()

The PySide2.QtXml.QDomElement.text() function operates recursively to find the text (since not all elements contain text). If you want to find all the text in all of a node’s children, iterate over the children looking for PySide2.QtXml.QDomText nodes, e.g.

text = QString()
element = doc.documentElement()

n = element.firstChild()
while True:
    if not n.isNull()
        break
    t = n.toText()
    if !t.isNull():
        text += t.data()

    n = n.nextSibling()

Note that we attempt to convert each node to a text node and use PySide2.QtXml.QDomElement.text() rather than using PySide2.QtXml.QDomNode.firstChild() . PySide2.QtXml.QDomNode.toText() . data() or n. PySide2.QtXml.QDomNode.toText() . data() directly on the node, because the node may not be a text element.

You can get a list of all the decendents of an element which have a specified tag name with PySide2.QtXml.QDomElement.elementsByTagName() or PySide2.QtXml.QDomElement.elementsByTagNameNS() .

To browse the elements of a dom document use PySide2.QtXml.QDomNode.firstChildElement() , PySide2.QtXml.QDomNode.lastChildElement() , PySide2.QtXml.QDomNode.nextSiblingElement() and PySide2.QtXml.QDomNode.previousSiblingElement() . For example, to iterate over all child elements called “entry” in a root element called “database”, you can use:

doc = # some QDomDocument ...
root = doc.firstChildElement("database")
elt = root.firstChildElement("entry")
while True:
    if not elt.isNull():
        break
    # ...
    elt = elt.nextSiblingElement("entry")

For further information about the Document Object Model see Level 1 and Level 2 Core . For a more general introduction of the DOM implementation see the PySide2.QtXml.QDomDocument documentation.

class PySide2.QtXml.QDomElement(x)

Constructs an empty element. Use the QDomDocument.createElement() function to construct elements with content.

PySide2.QtXml.QDomElement.hasAttribute(name)¶
Parameters:name – unicode
Return type:PySide2.QtCore.bool

Returns true if this element has an attribute called name ; otherwise returns false .

Note

This function does not take the presence of namespaces into account. As a result, the specified name will be tested against fully-qualified attribute names that include any namespace prefixes that may be present.

Use PySide2.QtXml.QDomElement.hasAttributeNS() to explicitly test for attributes with specific namespaces and names.

Parameters:

  • qName – unicode
  • value – unicode

Adds an attribute with the qualified name qName and the namespace URI nsURI with the value value . If an attribute with the same local name and namespace URI exists, its prefix is replaced by the prefix of qName and its value is repaced by value .

Although qName is the qualified name, the local name is used to decide if an existing attribute’s value should be replaced.

PySide2.QtXml.QDomElement.text()¶
Return type:unicode

Returns the element’s text or an empty string.

Example:

<h1>Hello <b>Qt</b> <![CDATA[<xml is cool>]]></h1>

The function of the PySide2.QtXml.QDomElement for the <h1> tag, will return the following text:

Hello Qt <xml is cool>

Comments are ignored by this function. It only evaluates PySide2.QtXml.QDomText and PySide2.QtXml.QDomCDATASection objects.
