Detailed Description¶

The PySide2.QtXml.QDomNode class is the base class for all the nodes in a DOM tree.

Many functions in the DOM return a PySide2.QtXml.QDomNode .

You can find out the type of a node using PySide2.QtXml.QDomNode.isAttr() , PySide2.QtXml.QDomNode.isCDATASection() , PySide2.QtXml.QDomNode.isDocumentFragment() , PySide2.QtXml.QDomNode.isDocument() , PySide2.QtXml.QDomNode.isDocumentType() , PySide2.QtXml.QDomNode.isElement() , PySide2.QtXml.QDomNode.isEntityReference() , PySide2.QtXml.QDomNode.isText() , PySide2.QtXml.QDomNode.isEntity() , PySide2.QtXml.QDomNode.isNotation() , PySide2.QtXml.QDomNode.isProcessingInstruction() , PySide2.QtXml.QDomNode.isCharacterData() and PySide2.QtXml.QDomNode.isComment() .

A PySide2.QtXml.QDomNode can be converted into one of its subclasses using PySide2.QtXml.QDomNode.toAttr() , PySide2.QtXml.QDomNode.toCDATASection() , PySide2.QtXml.QDomNode.toDocumentFragment() , PySide2.QtXml.QDomNode.toDocument() , PySide2.QtXml.QDomNode.toDocumentType() , PySide2.QtXml.QDomNode.toElement() , PySide2.QtXml.QDomNode.toEntityReference() , PySide2.QtXml.QDomNode.toText() , PySide2.QtXml.QDomNode.toEntity() , PySide2.QtXml.QDomNode.toNotation() , PySide2.QtXml.QDomNode.toProcessingInstruction() , PySide2.QtXml.QDomNode.toCharacterData() or PySide2.QtXml.QDomNode.toComment() . You can convert a node to a null node with PySide2.QtXml.QDomNode.clear() .

Copies of the PySide2.QtXml.QDomNode class share their data using explicit sharing. This means that modifying one node will change all copies. This is especially useful in combination with functions which return a PySide2.QtXml.QDomNode , e.g. PySide2.QtXml.QDomNode.firstChild() . You can make an independent (deep) copy of the node with PySide2.QtXml.QDomNode.cloneNode() .

A PySide2.QtXml.QDomNode can be null, much like a null pointer. Creating a copy of a null node results in another null node. It is not possible to modify a null node, but it is possible to assign another, possibly non-null node to it. In this case, the copy of the null node will remain null. You can check if a PySide2.QtXml.QDomNode is null by calling PySide2.QtXml.QDomNode.isNull() . The empty constructor of a PySide2.QtXml.QDomNode (or any of the derived classes) creates a null node.

Nodes are inserted with PySide2.QtXml.QDomNode.insertBefore() , PySide2.QtXml.QDomNode.insertAfter() or PySide2.QtXml.QDomNode.appendChild() . You can replace one node with another using PySide2.QtXml.QDomNode.replaceChild() and remove a node with PySide2.QtXml.QDomNode.removeChild() .

To traverse nodes use PySide2.QtXml.QDomNode.firstChild() to get a node’s first child (if any), and PySide2.QtXml.QDomNode.nextSibling() to traverse. PySide2.QtXml.QDomNode also provides PySide2.QtXml.QDomNode.lastChild() , PySide2.QtXml.QDomNode.previousSibling() and PySide2.QtXml.QDomNode.parentNode() . To find the first child node with a particular node name use PySide2.QtXml.QDomNode.namedItem() .

To find out if a node has children use PySide2.QtXml.QDomNode.hasChildNodes() and to get a list of all of a node’s children use PySide2.QtXml.QDomNode.childNodes() .

The node’s name and value (the meaning of which varies depending on its type) is returned by PySide2.QtXml.QDomNode.nodeName() and PySide2.QtXml.QDomNode.nodeValue() respectively. The node’s type is returned by PySide2.QtXml.QDomNode.nodeType() . The node’s value can be set with PySide2.QtXml.QDomNode.setNodeValue() .

The document to which the node belongs is returned by PySide2.QtXml.QDomNode.ownerDocument() .

Adjacent PySide2.QtXml.QDomText nodes can be merged into a single node with PySide2.QtXml.QDomNode.normalize() .

PySide2.QtXml.QDomElement nodes have attributes which can be retrieved with PySide2.QtXml.QDomNode.attributes() .

PySide2.QtXml.QDomElement and PySide2.QtXml.QDomAttr nodes can have namespaces which can be retrieved with PySide2.QtXml.QDomNode.namespaceURI() . Their local name is retrieved with PySide2.QtXml.QDomNode.localName() , and their prefix with PySide2.QtXml.QDomNode.prefix() . The prefix can be set with PySide2.QtXml.QDomNode.setPrefix() .

You can write the XML representation of the node to a text stream with PySide2.QtXml.QDomNode.save() .

The following example looks for the first element in an XML document and prints the names of all the elements that are its direct children.

d = QDomDocument()
d.setContent(someXML)
n = d.firstChild()
while !n.isNull():
    if n.isElement():
        e = n.toElement()
        print "Element name: %s" % e.tagName()
        break
    n = n.nextSibling()

For further information about the Document Object Model see Level 1 and Level 2 Core . For a more general introduction of the DOM implementation see the PySide2.QtXml.QDomDocument documentation.

PySide2.QtXml.QDomNode.appendChild(newChild)¶
Parameters:newChild – PySide2.QtXml.QDomNode
Return type:PySide2.QtXml.QDomNode

Appends newChild as the node’s last child.

If newChild is the child of another node, it is reparented to this node. If newChild is a child of this node, then its position in the list of children is changed.

If newChild is a PySide2.QtXml.QDomDocumentFragment , then the children of the fragment are removed from the fragment and appended.

If newChild is a PySide2.QtXml.QDomElement and this node is a PySide2.QtXml.QDomDocument that already has an element node as a child, newChild is not added as a child and a null node is returned.

Returns a new reference to newChild on success or a null node on failure.

Calling this function on a null node(created, for example, with the default constructor) does nothing and returns a null node .

The DOM specification disallow inserting attribute nodes, but for historical reasons, QDom accepts them anyway.

PySide2.QtXml.QDomNode.childNodes()¶
Return type:PySide2.QtXml.QDomNodeList

Returns a list of all direct child nodes.

Most often you will call this function on a PySide2.QtXml.QDomElement object.

For example, if the XML document looks like this:

<body>
<h1>Heading</h1>
<p>Hello <b>you</b></p>
</body>

Then the list of child nodes for the “body”-element will contain the node created by the &lt;h1&gt; tag and the node created by the &lt;p&gt; tag.

The nodes in the list are not copied; so changing the nodes in the list will also change the children of this node.

PySide2.QtXml.QDomNode.cloneNode([deep=true])¶
Parameters:deep – PySide2.QtCore.bool
Return type:PySide2.QtXml.QDomNode

Creates a deep (not shallow) copy of the PySide2.QtXml.QDomNode .

If deep is true, then the cloning is done recursively which means that all the node’s children are deep copied too. If deep is false only the node itself is copied and the copy will have no child nodes.

PySide2.QtXml.QDomNode.insertAfter(newChild, refChild)¶
Parameters:

  • refChild – PySide2.QtXml.QDomNode
Return type:

PySide2.QtXml.QDomNode

Inserts the node newChild after the child node refChild . refChild must be a direct child of this node. If refChild is null then newChild is appended as this node’s last child.

If newChild is the child of another node, it is reparented to this node. If newChild is a child of this node, then its position in the list of children is changed.

If newChild is a PySide2.QtXml.QDomDocumentFragment , then the children of the fragment are removed from the fragment and inserted after refChild .

Returns a new reference to newChild on success or a null node on failure.

The DOM specification disallow inserting attribute nodes, but due to historical reasons QDom accept them nevertheless.

The meaning of the name depends on the subclass:

Name Meaning

PySide2.QtXml.QDomCDATASection The string “#cdata-section”
PySide2.QtXml.QDomComment The string “#comment”
PySide2.QtXml.QDomDocument The string “#document”
PySide2.QtXml.QDomDocumentFragment The string “#document-fragment”
PySide2.QtXml.QDomDocumentType The name of the document type
PySide2.QtXml.QDomElement The tag name
PySide2.QtXml.QDomEntity The name of the entity
PySide2.QtXml.QDomEntityReference The name of the referenced entity
PySide2.QtXml.QDomNotation The name of the notation
PySide2.QtXml.QDomProcessingInstruction The target of the processing instruction
PySide2.QtXml.QDomText The string “#text”

Note

This function does not take the presence of namespaces into account when processing the names of element and attribute nodes. As a result, the returned name can contain any namespace prefix that may be present. To obtain the node name of an element or attribute, use PySide2.QtXml.QDomNode.localName() ; to obtain the namespace prefix, use PySide2.QtXml.QDomNode.namespaceURI() .

PySide2.QtXml.QDomNode.__eq__(arg__1)¶
Parameters:arg__1 – PySide2.QtXml.QDomNode
Return type:PySide2.QtCore.bool

Returns true if n and this DOM node are equal; otherwise returns false .

Any instance of PySide2.QtXml.QDomNode acts as a reference to an underlying data structure in PySide2.QtXml.QDomDocument . The test for equality checks if the two references point to the same underlying node. For example:

QDomDocument document
QDomElement element1 = document.documentElement()
QDomElement element2 = element1

The two nodes ( PySide2.QtXml.QDomElement is a PySide2.QtXml.QDomNode subclass) both refer to the document’s root element, and element1 == element2 will return true. On the other hand:

QDomElement element3 = document.createElement("MyElement")
QDomElement element4 = document.createElement("MyElement")

Even though both nodes are empty elements carrying the same name, element3 == element4 will return false because they refer to two different nodes in the underlying data structure.

PySide2.QtXml.QDomNode.prefix()¶
Return type:unicode

Returns the namespace prefix of the node or an empty string if the node has no namespace prefix.

Only nodes of type ElementNode or AttributeNode can have namespaces. A namespace prefix must be specified at creation time. If a node was created with a namespace prefix, you can change it later with PySide2.QtXml.QDomNode.setPrefix() .

If you create an element or attribute with QDomDocument.createElement() or QDomDocument.createAttribute() , the prefix will be an empty string. If you use QDomDocument.createElementNS() or QDomDocument.createAttributeNS() instead, the prefix will not be an empty string; but it might be an empty string if the name does not have a prefix.

PySide2.QtXml.QDomNode.save(arg__1, arg__2[, arg__3=QDomNode.EncodingFromDocument])¶
Parameters:
  • arg__1 – PySide2.QtCore.QTextStream
  • arg__2 – PySide2.QtCore.int
  • arg__3 – PySide2.QtXml.QDomNode.EncodingPolicy

Writes the XML representation of the node and all its children to the stream stream . This function uses indent as the amount of space to indent the node.

If the document contains invalid XML characters or characters that cannot be encoded in the given encoding, the result and behavior is undefined.

If encodingPolicy is QDomNode.EncodingFromDocument and this node is a document node, the encoding of text stream stream ‘s encoding is set by treating a processing instruction by name “xml” as an XML declaration, if one exists, and otherwise defaults to UTF-8. XML declarations are not processing instructions, but this behavior exists for historical reasons. If this node is not a document node, the text stream’s encoding is used.

If encodingPolicy is EncodingFromTextStream and this node is a document node, this function behaves as save( PySide2.QtCore.QTextStream &str, int indent) with the exception that the encoding specified in the text stream stream is used.

If the document contains invalid XML characters or characters that cannot be encoded in the given encoding, the result and behavior is undefined.
