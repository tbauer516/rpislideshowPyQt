Detailed Description¶

The PySide2.QtNetwork.QAbstractSocket class provides the base functionality common to all socket types.

PySide2.QtNetwork.QAbstractSocket is the base class for PySide2.QtNetwork.QTcpSocket and PySide2.QtNetwork.QUdpSocket and contains all common functionality of these two classes. If you need a socket, you have two options:

  • Instantiate PySide2.QtNetwork.QTcpSocket or PySide2.QtNetwork.QUdpSocket .
  • Create a native socket descriptor, instantiate PySide2.QtNetwork.QAbstractSocket , and call PySide2.QtNetwork.QAbstractSocket.setSocketDescriptor() to wrap the native socket.

TCP (Transmission Control Protocol) is a reliable, stream-oriented, connection-oriented transport protocol. UDP (User Datagram Protocol) is an unreliable, datagram-oriented, connectionless protocol. In practice, this means that TCP is better suited for continuous transmission of data, whereas the more lightweight UDP can be used when reliability isn’t important.

PySide2.QtNetwork.QAbstractSocket ‘s API unifies most of the differences between the two protocols. For example, although UDP is connectionless, PySide2.QtNetwork.QAbstractSocket.connectToHost() establishes a virtual connection for UDP sockets, enabling you to use PySide2.QtNetwork.QAbstractSocket in more or less the same way regardless of the underlying protocol. Internally, PySide2.QtNetwork.QAbstractSocket remembers the address and port passed to PySide2.QtNetwork.QAbstractSocket.connectToHost() , and functions like PySide2.QtCore.QIODevice.read() and PySide2.QtCore.QIODevice.write() use these values.

At any time, PySide2.QtNetwork.QAbstractSocket has a state (returned by PySide2.QtNetwork.QAbstractSocket.state() ). The initial state is UnconnectedState . After calling PySide2.QtNetwork.QAbstractSocket.connectToHost() , the socket first enters HostLookupState . If the host is found, PySide2.QtNetwork.QAbstractSocket enters ConnectingState and emits the PySide2.QtNetwork.QAbstractSocket.hostFound() signal. When the connection has been established, it enters ConnectedState and emits PySide2.QtNetwork.QAbstractSocket.connected() . If an error occurs at any stage, PySide2.QtNetwork.QAbstractSocket.error() is emitted. Whenever the state changes, PySide2.QtNetwork.QAbstractSocket.stateChanged() is emitted. For convenience, PySide2.QtNetwork.QAbstractSocket.isValid() returns true if the socket is ready for reading and writing, but note that the socket’s state must be ConnectedState before reading and writing can occur.

Read or write data by calling PySide2.QtCore.QIODevice.read() or PySide2.QtCore.QIODevice.write() , or use the convenience functions PySide2.QtCore.QIODevice.readLine() and PySide2.QtCore.QIODevice.readAll() . PySide2.QtNetwork.QAbstractSocket also inherits PySide2.QtCore.QIODevice.getChar() , PySide2.QtCore.QIODevice.putChar() , and PySide2.QtCore.QIODevice.ungetChar() from PySide2.QtCore.QIODevice , which work on single bytes. The PySide2.QtCore.QIODevice.bytesWritten() signal is emitted when data has been written to the socket. Note that Qt does not limit the write buffer size. You can monitor its size by listening to this signal.

The PySide2.QtCore.QIODevice.readyRead() signal is emitted every time a new chunk of data has arrived. PySide2.QtNetwork.QAbstractSocket.bytesAvailable() then returns the number of bytes that are available for reading. Typically, you would connect the PySide2.QtCore.QIODevice.readyRead() signal to a slot and read all available data there. If you don’t read all the data at once, the remaining data will still be available later, and any new incoming data will be appended to PySide2.QtNetwork.QAbstractSocket ‘s internal read buffer. To limit the size of the read buffer, call PySide2.QtNetwork.QAbstractSocket.setReadBufferSize() .

To close the socket, call PySide2.QtNetwork.QAbstractSocket.disconnectFromHost() . PySide2.QtNetwork.QAbstractSocket enters QAbstractSocket.ClosingState . After all pending data has been written to the socket, PySide2.QtNetwork.QAbstractSocket actually closes the socket, enters QAbstractSocket.UnconnectedState , and emits PySide2.QtNetwork.QAbstractSocket.disconnected() . If you want to abort a connection immediately, discarding all pending data, call PySide2.QtNetwork.QAbstractSocket.abort() instead. If the remote host closes the connection, PySide2.QtNetwork.QAbstractSocket will emit error( QAbstractSocket.RemoteHostClosedError ), during which the socket state will still be ConnectedState , and then the PySide2.QtNetwork.QAbstractSocket.disconnected() signal will be emitted.

The port and address of the connected peer is fetched by calling PySide2.QtNetwork.QAbstractSocket.peerPort() and PySide2.QtNetwork.QAbstractSocket.peerAddress() . PySide2.QtNetwork.QAbstractSocket.peerName() returns the host name of the peer, as passed to PySide2.QtNetwork.QAbstractSocket.connectToHost() . PySide2.QtNetwork.QAbstractSocket.localPort() and PySide2.QtNetwork.QAbstractSocket.localAddress() return the port and address of the local socket.

PySide2.QtNetwork.QAbstractSocket provides a set of functions that suspend the calling thread until certain signals are emitted. These functions can be used to implement blocking sockets:

  • PySide2.QtNetwork.QAbstractSocket.waitForConnected() blocks until a connection has been established.
  • PySide2.QtNetwork.QAbstractSocket.waitForReadyRead() blocks until new data is available for reading.
  • PySide2.QtNetwork.QAbstractSocket.waitForBytesWritten() blocks until one payload of data has been written to the socket.
  • PySide2.QtNetwork.QAbstractSocket.waitForDisconnected() blocks until the connection has closed.

We show an example:

numRead = 0
numReadTotal = 0

while(True):
    buffer  = socket.read(50)
    # do whatever with array
    numReadTotal += buffer.size()
    if (buffer.size() == 0 && !socket.waitForReadyRead()):
        break

If PySide2.QtCore.QIODevice.waitForReadyRead() returns false , the connection has been closed or an error has occurred.

Programming with a blocking socket is radically different from programming with a non-blocking socket. A blocking socket doesn’t require an event loop and typically leads to simpler code. However, in a GUI application, blocking sockets should only be used in non-GUI threads, to avoid freezing the user interface. See the fortuneclient and blockingfortuneclient examples for an overview of both approaches.

Note

We discourage the use of the blocking functions together with signals. One of the two possibilities should be used.

PySide2.QtNetwork.QAbstractSocket can be used with PySide2.QtCore.QTextStream and PySide2.QtCore.QDataStream ‘s stream operators (operator<<() and operator>>()). There is one issue to be aware of, though: You must make sure that enough data is available before attempting to read it using operator>>().

This enum describes the different flags you can pass to modify the behavior of QAbstractSocket.bind() .

Constant Description
QAbstractSocket.ShareAddress Allow other services to bind to the same address and port. This is useful when multiple processes share the load of a single service by listening to the same address and port (e.g., a web server with several pre-forked listeners can greatly improve response time). However, because any service is allowed to rebind, this option is subject to certain security considerations. Note that by combining this option with , you will also allow your service to rebind an existing shared address. On Unix, this is equivalent to the SO_REUSEADDR socket option. On Windows, this option is ignored.
QAbstractSocket.DontShareAddress Bind the address and port exclusively, so that no other services are allowed to rebind. By passing this option to QAbstractSocket.bind() , you are guaranteed that on successs, your service is the only one that listens to the address and port. No services are allowed to rebind, even if they pass . This option provides more security than , but on certain operating systems, it requires you to run the server with administrator privileges. PySide2.QtNetwork.QAbstractSocket.connectToHost(hostName, port[, mode=QIODevice.ReadWrite[, protocol=AnyIPProtocol]])¶
Parameters:
  • hostName – unicode
  • port – PySide2.QtCore.quint16
  • mode – PySide2.QtCore.QIODevice.OpenMode
  • protocol – PySide2.QtNetwork.QAbstractSocket.NetworkLayerProtocol

Attempts to make a connection to hostName on the given port . The protocol parameter can be used to specify which network protocol to use (eg. IPv4 or IPv6).

The socket is opened in the given openMode and first enters HostLookupState , then performs a host name lookup of hostName . If the lookup succeeds, PySide2.QtNetwork.QAbstractSocket.hostFound() is emitted and PySide2.QtNetwork.QAbstractSocket enters ConnectingState . It then attempts to connect to the address or addresses returned by the lookup. Finally, if a connection is established, PySide2.QtNetwork.QAbstractSocket enters ConnectedState and emits PySide2.QtNetwork.QAbstractSocket.connected() .

At any point, the socket can emit PySide2.QtNetwork.QAbstractSocket.error() to signal that an error occurred.

hostName may be an IP address in string form (e.g., “43.195.83.32”), or it may be a host name (e.g., “example.com”). PySide2.QtNetwork.QAbstractSocket will do a lookup only if required. port is in native byte order.

PySide2.QtNetwork.QAbstractSocket.flush()¶
Return type:PySide2.QtCore.bool

This function writes as much as possible from the internal write buffer to the underlying network socket, without blocking. If any data was written, this function returns true ; otherwise false is returned.

Call this function if you need PySide2.QtNetwork.QAbstractSocket to start sending buffered data immediately. The number of bytes successfully written depends on the operating system. In most cases, you do not need to call this function, because PySide2.QtNetwork.QAbstractSocket will start sending data automatically once control goes back to the event loop. In the absence of an event loop, call PySide2.QtNetwork.QAbstractSocket.waitForBytesWritten() instead.

If set to PauseOnSslErrors , data transfer on the socket will be paused and needs to be enabled explicitly again by calling PySide2.QtNetwork.QAbstractSocket.resume() . By default this option is set to PauseNever . This option must be called before connecting to the server, otherwise it will result in undefined behavior.

PySide2.QtNetwork.QAbstractSocket.setReadBufferSize(size)¶
Parameters:size – PySide2.QtCore.qint64

Sets the size of PySide2.QtNetwork.QAbstractSocket ‘s internal read buffer to be size bytes.

If the buffer size is limited to a certain size, PySide2.QtNetwork.QAbstractSocket won’t buffer more than this size of data. Exceptionally, a buffer size of 0 means that the read buffer is unlimited and all incoming data is buffered. This is the default.

This option is useful if you only read the data at certain points in time (e.g., in a real-time streaming application) or if you want to protect your socket against receiving too much data, which may eventually cause your application to run out of memory.

Only PySide2.QtNetwork.QTcpSocket uses PySide2.QtNetwork.QAbstractSocket ‘s internal buffer; PySide2.QtNetwork.QUdpSocket does not use any buffering at all, but rather relies on the implicit buffering provided by the operating system. Because of this, calling this function on PySide2.QtNetwork.QUdpSocket has no effect.

PySide2.QtNetwork.QAbstractSocket.waitForConnected([msecs=30000])¶
Parameters:msecs – PySide2.QtCore.int
Return type:PySide2.QtCore.bool

Waits until the socket is connected, up to msecs milliseconds. If the connection has been established, this function returns true ; otherwise it returns false . In the case where it returns false , you can call PySide2.QtNetwork.QAbstractSocket.error() to determine the cause of the error.

The following example waits up to one second for a connection to be established:

socket.connectToHost("imap", 143)
if socket.waitForConnected(1000):
    print "Connected!"

If msecs is -1, this function will not time out.

Note

This function may wait slightly longer than msecs , depending on the time it takes to complete the host lookup.

Note

Multiple calls to this functions do not accumulate the time. If the function times out, the connecting process will be aborted.

Note

This function may fail randomly on Windows. Consider using the event loop and the PySide2.QtNetwork.QAbstractSocket.connected() signal if your software will run on Windows.

See also

PySide2.QtNetwork.QAbstractSocket.connectToHost() PySide2.QtNetwork.QAbstractSocket.connected()
