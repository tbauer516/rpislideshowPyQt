Detailed Description¶

The PySide2.QtNetwork.QHostInfo class provides static functions for host name lookups.

PySide2.QtNetwork.QHostInfo uses the lookup mechanisms provided by the operating system to find the IP address(es) associated with a host name, or the host name associated with an IP address. The class provides two static convenience functions: one that works asynchronously and emits a signal once the host is found, and one that blocks and returns a PySide2.QtNetwork.QHostInfo object.

To look up a host’s IP addresses asynchronously, call PySide2.QtNetwork.QHostInfo.lookupHost() , which takes the host name or IP address, a receiver object, and a slot signature as arguments and returns an ID. You can abort the lookup by calling PySide2.QtNetwork.QHostInfo.abortHostLookup() with the lookup ID.

Example:

# To find the IP address of qt-project.org
QHostInfo.lookupHost("qt-project.org", self, SLOT("printResults(QHostInfo)"))

# To find the host name for 4.2.2.1
QHostInfo.lookupHost("4.2.2.1", self, SLOT("printResults(QHostInfo)"))

The slot is invoked when the results are ready. The results are stored in a PySide2.QtNetwork.QHostInfo object. Call PySide2.QtNetwork.QHostInfo.addresses() to get the list of IP addresses for the host, and PySide2.QtNetwork.QHostInfo.hostName() to get the host name that was looked up.

If the lookup failed, PySide2.QtNetwork.QHostInfo.error() returns the type of error that occurred. PySide2.QtNetwork.QHostInfo.errorString() gives a human-readable description of the lookup error.

If you want a blocking lookup, use the QHostInfo.fromName() function:

info = QHostInfo.fromName("qt-project.org")

PySide2.QtNetwork.QHostInfo supports Internationalized Domain Names (IDNs) through the IDNA and Punycode standards.

To retrieve the name of the local host, use the static QHostInfo.localHostName() function.

Note

Since Qt 4.6.1 PySide2.QtNetwork.QHostInfo is using multiple threads for DNS lookup instead of one dedicated DNS thread. This improves performance, but also changes the order of signal emissions when using PySide2.QtNetwork.QHostInfo.lookupHost() compared to previous versions of Qt.

Note

Since Qt 4.6.3 PySide2.QtNetwork.QHostInfo is using a small internal 60 second DNS cache for performance improvements.

PySide2.QtNetwork.QHostInfo.HostInfoError¶

This enum describes the various errors that can occur when trying to resolve a host name.

Constant Description
QHostInfo.NoError The lookup was successful.
QHostInfo.HostNotFound No IP addresses were found for the host.
QHostInfo.UnknownError An unknown error occurred.

static PySide2.QtNetwork.QHostInfo.fromName(name)¶
Parameters:name – unicode
Return type:PySide2.QtNetwork.QHostInfo

Looks up the IP address(es) for the given host name . The function blocks during the lookup which means that execution of the program is suspended until the results of the lookup are ready. Returns the result of the lookup in a PySide2.QtNetwork.QHostInfo object.

If you pass a literal IP address to name instead of a host name, PySide2.QtNetwork.QHostInfo will search for the domain name for the IP (i.e., PySide2.QtNetwork.QHostInfo will perform a reverse lookup). On success, the returned PySide2.QtNetwork.QHostInfo will contain both the resolved domain name and IP addresses for the host name.

static PySide2.QtNetwork.QHostInfo.localHostName()¶
Return type:unicode

Returns this machine’s host name, if one is configured. Note that hostnames are not guaranteed to be globally unique, especially if they were configured automatically.

This function does not guarantee the returned host name is a Fully Qualified Domain Name (FQDN). For that, use PySide2.QtNetwork.QHostInfo.fromName() to resolve the returned name to an FQDN.

This function returns the same as QSysInfo.machineHostName() .
