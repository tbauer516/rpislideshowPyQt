Detailed Description¶

The PySide2.QtNetwork.QTcpServer class provides a TCP-based server.

This class makes it possible to accept incoming TCP connections. You can specify the port or have PySide2.QtNetwork.QTcpServer pick one automatically. You can listen on a specific address or on all the machine’s addresses.

Call PySide2.QtNetwork.QTcpServer.listen() to have the server listen for incoming connections. The PySide2.QtNetwork.QTcpServer.newConnection() signal is then emitted each time a client connects to the server.

Call PySide2.QtNetwork.QTcpServer.nextPendingConnection() to accept the pending connection as a connected PySide2.QtNetwork.QTcpSocket . The function returns a pointer to a PySide2.QtNetwork.QTcpSocket in QAbstractSocket.ConnectedState that you can use for communicating with the client.

If an error occurs, PySide2.QtNetwork.QTcpServer.serverError() returns the type of error, and PySide2.QtNetwork.QTcpServer.errorString() can be called to get a human readable description of what happened.

When listening for connections, the address and port on which the server is listening are available as PySide2.QtNetwork.QTcpServer.serverAddress() and PySide2.QtNetwork.QTcpServer.serverPort() .

Calling PySide2.QtNetwork.QTcpServer.close() makes PySide2.QtNetwork.QTcpServer stop listening for incoming connections.

Although PySide2.QtNetwork.QTcpServer is mostly designed for use with an event loop, it’s possible to use it without one. In that case, you must use PySide2.QtNetwork.QTcpServer.waitForNewConnection() , which blocks until either a connection is available or a timeout expires.

PySide2.QtNetwork.QTcpServer.incomingConnection(handle)¶
Parameters:handle – PySide2.QtCore.qintptr

This virtual function is called by PySide2.QtNetwork.QTcpServer when a new connection is available. The socketDescriptor argument is the native socket descriptor for the accepted connection.

The base implementation creates a PySide2.QtNetwork.QTcpSocket , sets the socket descriptor and then stores the PySide2.QtNetwork.QTcpSocket in an internal list of pending connections. Finally PySide2.QtNetwork.QTcpServer.newConnection() is emitted.

Reimplement this function to alter the server’s behavior when a connection is available.

If this server is using PySide2.QtNetwork.QNetworkProxy then the socketDescriptor may not be usable with native socket functions, and should only be used with QTcpSocket.setSocketDescriptor() .

Note

If another socket is created in the reimplementation of this method, it needs to be added to the Pending Connections mechanism by calling PySide2.QtNetwork.QTcpServer.addPendingConnection() .

Note

If you want to handle an incoming connection as a new PySide2.QtNetwork.QTcpSocket object in another thread you have to pass the PySide2.QtNetwork.QTcpServer.socketDescriptor() to the other thread and create the PySide2.QtNetwork.QTcpSocket object there and use its PySide2.QtNetwork.QTcpServer.setSocketDescriptor() method.

PySide2.QtNetwork.QTcpServer.nextPendingConnection()¶
Return type:PySide2.QtNetwork.QTcpSocket

Returns the next pending connection as a connected PySide2.QtNetwork.QTcpSocket object.

The socket is created as a child of the server, which means that it is automatically deleted when the PySide2.QtNetwork.QTcpServer object is destroyed. It is still a good idea to delete the object explicitly when you are done with it, to avoid wasting memory.

0 is returned if this function is called when there are no pending connections.

Note

The returned PySide2.QtNetwork.QTcpSocket object cannot be used from another thread. If you want to use an incoming connection from another thread, you need to override PySide2.QtNetwork.QTcpServer.incomingConnection() .

PySide2.QtNetwork.QTcpServer.setMaxPendingConnections(numConnections)¶
Parameters:numConnections – PySide2.QtCore.int

Sets the maximum number of pending accepted connections to numConnections . PySide2.QtNetwork.QTcpServer will accept no more than numConnections incoming connections before PySide2.QtNetwork.QTcpServer.nextPendingConnection() is called. By default, the limit is 30 pending connections.

Clients may still able to connect after the server has reached its maximum number of pending connections (i.e., PySide2.QtNetwork.QTcpSocket can still emit the connected() signal). PySide2.QtNetwork.QTcpServer will stop accepting the new connections, but the operating system may still keep them in queue.

PySide2.QtNetwork.QTcpServer.waitForNewConnection(msec)¶
Parameters:msec – PySide2.QtCore.int
Return type:(retval, timeOut)

Waits for at most msec milliseconds or until an incoming connection is available. Returns true if a connection is available; otherwise returns false . If the operation timed out and timedOut is not 0, *``timedOut`` will be set to true.

This is a blocking function call. Its use is disadvised in a single-threaded GUI application, since the whole application will stop responding until the function returns. is mostly useful when there is no event loop available.

The non-blocking alternative is to connect to the PySide2.QtNetwork.QTcpServer.newConnection() signal.

If msec is -1, this function will not time out.
