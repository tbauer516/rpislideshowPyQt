Detailed Description¶

The PySide2.QtNetwork.QUdpSocket class provides a UDP socket.

UDP (User Datagram Protocol) is a lightweight, unreliable, datagram-oriented, connectionless protocol. It can be used when reliability isn’t important. PySide2.QtNetwork.QUdpSocket is a subclass of PySide2.QtNetwork.QAbstractSocket that allows you to send and receive UDP datagrams.

The most common way to use this class is to bind to an address and port using PySide2.QtNetwork.QAbstractSocket.bind() , then call PySide2.QtNetwork.QUdpSocket.writeDatagram() and PySide2.QtNetwork.QUdpSocket.readDatagram() / PySide2.QtNetwork.QUdpSocket.receiveDatagram() to transfer data. If you want to use the standard PySide2.QtCore.QIODevice functions PySide2.QtCore.QIODevice.read() , PySide2.QtCore.QIODevice.readLine() , PySide2.QtCore.QIODevice.write() , etc., you must first connect the socket directly to a peer by calling PySide2.QtNetwork.QAbstractSocket.connectToHost() .

The socket emits the PySide2.QtCore.QIODevice.bytesWritten() signal every time a datagram is written to the network. If you just want to send datagrams, you don’t need to call PySide2.QtNetwork.QAbstractSocket.bind() .

The PySide2.QtCore.QIODevice.readyRead() signal is emitted whenever datagrams arrive. In that case, PySide2.QtNetwork.QUdpSocket.hasPendingDatagrams() returns true . Call PySide2.QtNetwork.QUdpSocket.pendingDatagramSize() to obtain the size of the first pending datagram, and PySide2.QtNetwork.QUdpSocket.readDatagram() or PySide2.QtNetwork.QUdpSocket.receiveDatagram() to read it.

Note

An incoming datagram should be read when you receive the PySide2.QtCore.QIODevice.readyRead() signal, otherwise this signal will not be emitted for the next datagram.

Example:

def initSocket(self):
    udpSocket = QUdpSocket(self)
    udpSocket.bind(QHostAddress.LocalHost, 7755)

    self.connect(udpSocket, SIGNAL('readyRead()'),
                 self, SLOT('readPendingDatagrams()'))

def readPendingDatagrams(self):
    while udpSocket.hasPendingDatagrams():
        datagram = QByteArray()
        datagram.resize(udpSocket.pendingDatagramSize())

        (sender, senderPort) = udpSocket.readDatagram(datagram.data(), datagram.size())

        processTheDatagram(datagram)

PySide2.QtNetwork.QUdpSocket also supports UDP multicast. Use PySide2.QtNetwork.QUdpSocket.joinMulticastGroup() and PySide2.QtNetwork.QUdpSocket.leaveMulticastGroup() to control group membership, and QAbstractSocket.MulticastTtlOption and QAbstractSocket.MulticastLoopbackOption to set the TTL and loopback socket options. Use PySide2.QtNetwork.QUdpSocket.setMulticastInterface() to control the outgoing interface for multicast datagrams, and PySide2.QtNetwork.QUdpSocket.multicastInterface() to query it.

With PySide2.QtNetwork.QUdpSocket , you can also establish a virtual connection to a UDP server using PySide2.QtNetwork.QAbstractSocket.connectToHost() and then use PySide2.QtCore.QIODevice.read() and PySide2.QtCore.QIODevice.write() to exchange datagrams without specifying the receiver for each datagram.

The Broadcast Sender , Broadcast Receiver , Multicast Sender , and Multicast Receiver examples illustrate how to use PySide2.QtNetwork.QUdpSocket in applications.

PySide2.QtNetwork.QUdpSocket.joinMulticastGroup(groupAddress)¶
Parameters:groupAddress – PySide2.QtNetwork.QHostAddress
Return type:PySide2.QtCore.bool

Joins the multicast group specified by groupAddress on the default interface chosen by the operating system. The socket must be in BoundState , otherwise an error occurs.

Note that if you are attempting to join an IPv4 group, your socket must not be bound using IPv6 (or in dual mode, using QHostAddress.Any ). You must use QHostAddress.AnyIPv4 instead.

This function returns true if successful; otherwise it returns false and sets the socket error accordingly.

Note

Joining IPv6 multicast groups without an interface selection is not supported in all operating systems. Consider using the overload where the interface is specified.

PySide2.QtNetwork.QUdpSocket.multicastInterface()¶
Return type:PySide2.QtNetwork.QNetworkInterface

Returns the interface for the outgoing interface for multicast datagrams. This corresponds to the IP_MULTICAST_IF socket option for IPv4 sockets and the IPV6_MULTICAST_IF socket option for IPv6 sockets. If no interface has been previously set, this function returns an invalid PySide2.QtNetwork.QNetworkInterface . The socket must be in BoundState , otherwise an invalid PySide2.QtNetwork.QNetworkInterface is returned.

PySide2.QtNetwork.QUdpSocket.readDatagram(maxlen)¶
Parameters:maxlen – PySide2.QtCore.qint64
Return type:(data, address, port)

Receives a datagram no larger than maxSize bytes and stores it in data . The sender’s host address and port is stored in *``address`` and *``port`` (unless the pointers are 0).

Returns the size of the datagram on success; otherwise returns -1.

If maxSize is too small, the rest of the datagram will be lost. To avoid loss of data, call PySide2.QtNetwork.QUdpSocket.pendingDatagramSize() to determine the size of the pending datagram before attempting to read it. If maxSize is 0, the datagram will be discarded.

PySide2.QtNetwork.QUdpSocket.receiveDatagram([maxSize=-1])¶
Parameters:maxSize – PySide2.QtCore.qint64
Return type:PySide2.QtNetwork.QNetworkDatagram

Receives a datagram no larger than maxSize bytes and returns it in the PySide2.QtNetwork.QNetworkDatagram object, along with the sender’s host address and port. If possible, this function will also try to determine the datagram’s destination address, port, and the number of hop counts at reception time.

On failure, returns a PySide2.QtNetwork.QNetworkDatagram that reports not valid .

If maxSize is too small, the rest of the datagram will be lost. If maxSize is 0, the datagram will be discarded. If maxSize is -1 (the default), this function will attempt to read the entire datagram.

PySide2.QtNetwork.QUdpSocket.writeDatagram(datagram)
Parameters:datagram – PySide2.QtNetwork.QNetworkDatagram
Return type:PySide2.QtCore.qint64

This is an overloaded function.

Sends the datagram datagram to the host address and port numbers contained in datagram , using the network interface and hop count limits also set there. If the destination address and port numbers are unset, this function will send to the address that was passed to PySide2.QtNetwork.QAbstractSocket.connectToHost() .

If the destination address is IPv6 with a non-empty scope id but differs from the interface index in datagram , it is undefined which interface the operating system will choose to send on.

The function returns the number of bytes sent if it succeeded or -1 if it encountered an error.

Warning

Calling this function on a connected UDP socket may result in an error and no packet being sent. If you are using a connected socket, use PySide2.QtCore.QIODevice.write() to send datagrams.
