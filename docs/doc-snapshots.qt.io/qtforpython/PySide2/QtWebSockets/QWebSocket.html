Detailed Description¶

Implements a TCP socket that talks the protocol.

is a web technology providing full-duplex communications channels over a single TCP connection. The protocol was standardized by the IETF as RFC 6455 in 2011. PySide2.QtWebSockets.QWebSocket can both be used in a client application and server application.

This class was modeled after PySide2.QtNetwork.QAbstractSocket .

PySide2.QtWebSockets.QWebSocket currently does not support WebSocket Extensions and WebSocket Subprotocols .

PySide2.QtWebSockets.QWebSocket only supports version 13 of the protocol, as outlined in RFC 6455 .

Note

Some proxies do not understand certain HTTP headers used during a handshake. In that case, non-secure connections fail. The best way to mitigate against this problem is to use over a secure connection.

Warning

To generate masks, this implementation of uses the reasonably secure QRandomGenerator.global() ->generate() function. For more information about the importance of good masking, see “Talking to Yourself for Fun and Profit” by Lin-Shung Huang et al . The best measure against attacks mentioned in the document above, is to use PySide2.QtWebSockets.QWebSocket over a secure connection (wss:// ). In general, always be careful to not have 3rd party script access to a PySide2.QtWebSockets.QWebSocket in your application.

class PySide2.QtWebSockets.QWebSocket([origin=""[, version=QWebSocketProtocol.VersionLatest[, parent=nullptr]]])¶
Parameters:
  • origin – unicode
  • version – PySide2.QtWebSockets.QWebSocketProtocol.Version
  • parent – PySide2.QtCore.QObject

Creates a new PySide2.QtWebSockets.QWebSocket with the given origin , the version of the protocol to use and parent .

The origin of the client is as specified in RFC 6454 . (The origin is not required for non-web browser clients (see RFC 6455 )). The origin may not contain new line characters, otherwise the connection will be aborted immediately during the handshake phase.

Note

Currently only V13 ( RFC 6455 ) is supported

PySide2.QtWebSockets.QWebSocket.abort()¶

Aborts the current socket and resets the socket. Unlike PySide2.QtWebSockets.QWebSocket.close() , this function immediately closes the socket, discarding any pending data in the write buffer.

PySide2.QtWebSockets.QWebSocket.flush()¶
Return type:PySide2.QtCore.bool

This function writes as much as possible from the internal write buffer to the underlying network socket, without blocking. If any data was written, this function returns true; otherwise false is returned. Call this function if you need PySide2.QtWebSockets.QWebSocket to start sending buffered data immediately. The number of bytes successfully written depends on the operating system. In most cases, you do not need to call this function, because PySide2.QtWebSockets.QWebSocket will start sending data automatically once control goes back to the event loop.

PySide2.QtWebSockets.QWebSocket.resume()¶

Continues data transfer on the socket. This method should only be used after the socket has been set to pause upon notifications and a notification has been received. The only notification currently supported is sslErrors() . Calling this method if the socket is not paused results in undefined behavior.

PySide2.QtWebSockets.QWebSocket.setPauseMode(pauseMode)¶
Parameters:pauseMode – PySide2.QtNetwork.QAbstractSocket.PauseModes

Controls whether to pause upon receiving a notification. The pauseMode parameter specifies the conditions in which the socket should be paused.

The only notification currently supported is sslErrors() . If set to PauseOnSslErrors, data transfer on the socket will be paused and needs to be enabled explicitly again by calling PySide2.QtWebSockets.QWebSocket.resume() . By default, this option is set to PauseNever. This option must be called before connecting to the server, otherwise it will result in undefined behavior.

PySide2.QtWebSockets.QWebSocket.setReadBufferSize(size)¶
Parameters:size – PySide2.QtCore.qint64

Sets the size of PySide2.QtWebSockets.QWebSocket ‘s internal read buffer to be size bytes.

If the buffer size is limited to a certain size, PySide2.QtWebSockets.QWebSocket won’t buffer more than this size of data. Exceptionally, a buffer size of 0 means that the read buffer is unlimited and all incoming data is buffered. This is the default. This option is useful if you only read the data at certain points in time (for example, in a real-time streaming application) or if you want to protect your socket against receiving too much data, which may eventually cause your application to run out of memory.
