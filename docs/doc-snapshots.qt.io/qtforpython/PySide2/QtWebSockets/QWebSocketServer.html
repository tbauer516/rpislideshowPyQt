Detailed Description¶

Implements a -based server.

It is modeled after PySide2.QtNetwork.QTcpServer , and behaves the same. So, if you know how to use PySide2.QtNetwork.QTcpServer , you know how to use PySide2.QtWebSockets.QWebSocketServer . This class makes it possible to accept incoming connections. You can specify the port or have PySide2.QtWebSockets.QWebSocketServer pick one automatically. You can listen on a specific address or on all the machine’s addresses. Call PySide2.QtWebSockets.QWebSocketServer.listen() to have the server listen for incoming connections.

The PySide2.QtWebSockets.QWebSocketServer.newConnection() signal is then emitted each time a client connects to the server. Call PySide2.QtWebSockets.QWebSocketServer.nextPendingConnection() to accept the pending connection as a connected PySide2.QtWebSockets.QWebSocket . The function returns a pointer to a PySide2.QtWebSockets.QWebSocket in QAbstractSocket.ConnectedState that you can use for communicating with the client.

If an error occurs, PySide2.QtWebSockets.QWebSocketServer.serverError() returns the type of error, and PySide2.QtWebSockets.QWebSocketServer.errorString() can be called to get a human readable description of what happened.

When listening for connections, the address and port on which the server is listening are available as PySide2.QtWebSockets.QWebSocketServer.serverAddress() and PySide2.QtWebSockets.QWebSocketServer.serverPort() .

Calling PySide2.QtWebSockets.QWebSocketServer.close() makes PySide2.QtWebSockets.QWebSocketServer stop listening for incoming connections.

PySide2.QtWebSockets.QWebSocketServer currently does not support WebSocket Extensions and WebSocket Subprotocols .

Note

When working with self-signed certificates, Firefox bug 594502 prevents Firefox to connect to a secure server. To work around this problem, first browse to the secure server using HTTPS. FireFox will indicate that the certificate is invalid. From here on, the certificate can be added to the exceptions. After this, the secure connection should work.

PySide2.QtWebSockets.QWebSocketServer only supports version 13 of the protocol, as outlined in RFC 6455 .

class PySide2.QtWebSockets.QWebSocketServer(serverName, secureMode[, parent=nullptr])¶
Parameters:
  • secureMode – PySide2.QtWebSockets.QWebSocketServer.SslMode
  • serverName – unicode
  • parent – PySide2.QtCore.QObject

Constructs a new PySide2.QtWebSockets.QWebSocketServer with the given serverName . The serverName will be used in the HTTP handshake phase to identify the server. It can be empty, in which case no server name will be sent to the client. The secureMode parameter indicates whether the server operates over wss ( SecureMode ) or over ws ( NonSecureMode ).

parent is passed to the PySide2.QtCore.QObject constructor.

PySide2.QtWebSockets.QWebSocketServer.SslMode¶

Indicates whether the server operates over wss () or ws ()

Constant Description
QWebSocketServer.SecureMode The server operates in secure mode (over wss)
QWebSocketServer.NonSecureMode The server operates in non-secure mode (over ws)

PySide2.QtWebSockets.QWebSocketServer.nextPendingConnection()¶
Return type:PySide2.QtWebSockets.QWebSocket

Returns the next pending connection as a connected PySide2.QtWebSockets.QWebSocket object. PySide2.QtWebSockets.QWebSocketServer does not take ownership of the returned PySide2.QtWebSockets.QWebSocket object. It is up to the caller to delete the object explicitly when it will no longer be used, otherwise a memory leak will occur. nullptr is returned if this function is called when there are no pending connections.

Note: The returned PySide2.QtWebSockets.QWebSocket object cannot be used from another thread.

PySide2.QtWebSockets.QWebSocketServer.setMaxPendingConnections(numConnections)¶
Parameters:numConnections – PySide2.QtCore.int

Sets the maximum number of pending accepted connections to numConnections . WebSocketServer will accept no more than numConnections incoming connections before PySide2.QtWebSockets.QWebSocketServer.nextPendingConnection() is called. By default, the limit is 30 pending connections.

PySide2.QtWebSockets.QWebSocketServer will emit the PySide2.QtWebSockets.QWebSocketServer.error() signal with the QWebSocketProtocol.CloseCodeAbnormalDisconnection close code when the maximum of connections has been reached. The handshake will fail and the socket will be closed.
