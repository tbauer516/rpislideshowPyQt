Detailed Description¶

The PySide2.QtMultimedia.QAudioBuffer class represents a collection of audio samples with a specific format and sample rate.
class PySide2.QtMultimedia.QAudioBuffer¶
class PySide2.QtMultimedia.QAudioBuffer(other)
class PySide2.QtMultimedia.QAudioBuffer(data, format[, startTime=-1])
class PySide2.QtMultimedia.QAudioBuffer(numFrames, format[, startTime=-1])
Parameters:
  • data – PySide2.QtCore.QByteArray
  • format – PySide2.QtMultimedia.QAudioFormat
  • numFrames – PySide2.QtCore.int
  • startTime – PySide2.QtCore.qint64
  • other – PySide2.QtMultimedia.QAudioBuffer

Create a new, empty, invalid buffer.

Creates a new audio buffer from other . Generally this will have copy-on-write semantics - a copy will only be made when it has to be.

Creates a new audio buffer from the supplied data , in the given format . The format will determine how the number and sizes of the samples are interpreted from the data .

If the supplied data is not an integer multiple of the calculated frame size, the excess data will not be used.

This audio buffer will copy the contents of data .

startTime (in microseconds) indicates when this buffer starts in the stream. If this buffer is not part of a stream, set it to -1.

Creates a new audio buffer with space for numFrames frames of the given format . The individual samples will be initialized to the default for the format.

startTime (in microseconds) indicates when this buffer starts in the stream. If this buffer is not part of a stream, set it to -1.

PySide2.QtMultimedia.QAudioBuffer.byteCount()¶
Return type:PySide2.QtCore.int

Returns the size of this buffer, in bytes.

PySide2.QtMultimedia.QAudioBuffer.constData()¶
Return type:void

Returns a pointer to this buffer’s data. You can only read it.

This method is preferred over the const version of PySide2.QtMultimedia.QAudioBuffer.data() to prevent unnecessary copying.

There is also a templatized version of this function that allows you to retrieve a specific type of read-only pointer to the data. Note that there is no checking done on the format of the audio buffer - this is simply a convenience function.

// With a 16bit sample buffer:
const quint16 *data = buffer->constData<quint16>();
PySide2.QtMultimedia.QAudioBuffer.data()¶
Return type:void

Returns a pointer to this buffer’s data. You can only read it.

You should use the PySide2.QtMultimedia.QAudioBuffer.constData() function rather than this to prevent accidental deep copying.

There is also a templatized version of this function that allows you to retrieve a specific type of read-only pointer to the data. Note that there is no checking done on the format of the audio buffer - this is simply a convenience function.

// With a 16bit sample const buffer:
const quint16 *data = buffer->data<quint16>();
PySide2.QtMultimedia.QAudioBuffer.duration()¶
Return type:PySide2.QtCore.qint64

Returns the duration of audio in this buffer, in microseconds.

This depends on the PySide2.QtMultimedia.QAudioBuffer.format() , and the PySide2.QtMultimedia.QAudioBuffer.frameCount() .

PySide2.QtMultimedia.QAudioBuffer.format()¶
Return type:PySide2.QtMultimedia.QAudioFormat

Returns the format of this buffer.

Several properties of this format influence how the PySide2.QtMultimedia.QAudioBuffer.duration() or PySide2.QtMultimedia.QAudioBuffer.byteCount() are calculated from the PySide2.QtMultimedia.QAudioBuffer.frameCount() .

PySide2.QtMultimedia.QAudioBuffer.frameCount()¶
Return type:PySide2.QtCore.int

Returns the number of complete audio frames in this buffer.

An audio frame is an interleaved set of one sample per channel for the same instant in time.

PySide2.QtMultimedia.QAudioBuffer.isValid()¶
Return type:PySide2.QtCore.bool

Returns true if this is a valid buffer. A valid buffer has more than zero frames in it and a valid format.

PySide2.QtMultimedia.QAudioBuffer.sampleCount()¶
Return type:PySide2.QtCore.int

Returns the number of samples in this buffer.

If the format of this buffer has multiple channels, then this count includes all channels. This means that a stereo buffer with 1000 samples in total will have 500 left samples and 500 right samples (interleaved), and this function will return 1000.

See also

PySide2.QtMultimedia.QAudioBuffer.frameCount()

PySide2.QtMultimedia.QAudioBuffer.startTime()¶
Return type:PySide2.QtCore.qint64

Returns the time in a stream that this buffer starts at (in microseconds).

If this buffer is not part of a stream, this will return -1.
