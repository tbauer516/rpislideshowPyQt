Detailed Description¶

The PySide2.QtCore.QVariantAnimation class provides a base class for animations.

This class is part of The Animation Framework . It serves as a base class for property and item animations, with functions for shared functionality.

The class performs interpolation over PySide2.QtCore.QVariant s, but leaves using the interpolated values to its subclasses. Currently, Qt provides PySide2.QtCore.QPropertyAnimation , which animates Qt properties . See the PySide2.QtCore.QPropertyAnimation class description if you wish to animate such properties.

You can then set start and end values for the property by calling PySide2.QtCore.QVariantAnimation.setStartValue() and PySide2.QtCore.QVariantAnimation.setEndValue() , and finally call PySide2.QtCore.QAbstractAnimation.start() to start the animation. PySide2.QtCore.QVariantAnimation will interpolate the property of the target object and emit PySide2.QtCore.QVariantAnimation.valueChanged() . To react to a change in the current value you have to reimplement the PySide2.QtCore.QVariantAnimation.updateCurrentValue() virtual function or connect to said signal.

It is also possible to set values at specified steps situated between the start and end value. The interpolation will then touch these points at the specified steps. Note that the start and end values are defined as the key values at 0.0 and 1.0.

There are two ways to affect how PySide2.QtCore.QVariantAnimation interpolates the values. You can set an easing curve by calling PySide2.QtCore.QVariantAnimation.setEasingCurve() , and configure the duration by calling PySide2.QtCore.QVariantAnimation.setDuration() . You can change how the PySide2.QtCore.QVariant s are interpolated by creating a subclass of PySide2.QtCore.QVariantAnimation , and reimplementing the virtual PySide2.QtCore.QVariantAnimation.interpolated() function.

Subclassing PySide2.QtCore.QVariantAnimation can be an alternative if you have PySide2.QtCore.QVariant s that you do not wish to declare as Qt properties. Note, however, that you in most cases will be better off declaring your PySide2.QtCore.QVariant as a property.

Not all PySide2.QtCore.QVariant types are supported. Below is a list of currently supported PySide2.QtCore.QVariant types:

  • Int
  • UInt
  • Double
  • Float
  • QLine
  • QLineF
  • QPoint
  • QPointF
  • QSize
  • QSizeF
  • QRect
  • QRectF
  • QColor

If you need to interpolate other variant types, including custom types, you have to implement interpolation for these yourself. To do this, you can register an interpolator function for a given type. This function takes 3 parameters: the start value, the end value, and the current progress.

Example:

QVariant myColorInterpolator(const QColor &start, const QColor &end, qreal progress)
{
    ...
    return QColor(...);
}
...
qRegisterAnimationInterpolator<QColor>(myColorInterpolator);

Another option is to reimplement PySide2.QtCore.QVariantAnimation.interpolated() , which returns interpolation values for the value being interpolated.

PySide2.QtCore.QVariantAnimation.interpolated(from, to, progress)¶
Parameters:
  • from – object
  • to – object
  • progress – PySide2.QtCore.qreal
Return type:

object

This virtual function returns the linear interpolation between variants from and to , at progress , usually a value between 0 and 1. You can reimplement this function in a subclass of PySide2.QtCore.QVariantAnimation to provide your own interpolation algorithm.

Note that in order for the interpolation to work with a PySide2.QtCore.QEasingCurve that return a value smaller than 0 or larger than 1 (such as QEasingCurve.InBack ) you should make sure that it can extrapolate. If the semantic of the datatype does not allow extrapolation this function should handle that gracefully.

You should call the PySide2.QtCore.QVariantAnimation implementation of this function if you want your class to handle the types already supported by Qt (see class PySide2.QtCore.QVariantAnimation description for a list of supported types).

PySide2.QtCore.QVariantAnimation.updateCurrentValue(value)¶
Parameters:value – object

This virtual function is called every time the animation’s current value changes. The value argument is the new current value.

The base class implementation does nothing.
