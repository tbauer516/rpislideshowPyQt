Detailed Description¶

The PySide2.QtSql.QSqlTableModel class provides an editable data model for a single database table.

PySide2.QtSql.QSqlTableModel is a high-level interface for reading and writing database records from a single table. It is built on top of the lower-level PySide2.QtSql.QSqlQuery and can be used to provide data to view classes such as PySide2.QtWidgets.QTableView . For example:

model =  QSqlTableModel()
model.setTable("employee")
model.setEditStrategy(QSqlTableModel.OnManualSubmit)
model.select()
model.removeColumn(0) # don't show the ID
model.setHeaderData(0, Qt.Horizontal, tr("Name"))
model.setHeaderData(1, Qt.Horizontal, tr("Salary"))

view =  QTableView()
view.setModel(model)
view.show()

We set the SQL table’s name and the edit strategy, then we set up the labels displayed in the view header. The edit strategy dictates when the changes done by the user in the view are actually applied to the database. The possible values are OnFieldChange , OnRowChange , and OnManualSubmit .

PySide2.QtSql.QSqlTableModel can also be used to access a database programmatically, without binding it to a view:

model = QSqlQueryModel()
model.setQuery("SELECT * FROM employee")
salary = model.record(4).value("salary")

The code snippet above extracts the salary field from record 4 in the result set of the query SELECT * from employee .

It is possible to set filters using PySide2.QtSql.QSqlTableModel.setFilter() , or modify the sort order using PySide2.QtSql.QSqlTableModel.setSort() . At the end, you must call PySide2.QtSql.QSqlTableModel.select() to populate the model with data.

The tablemodel example illustrates how to use PySide2.QtSql.QSqlTableModel as the data source for a PySide2.QtWidgets.QTableView .

PySide2.QtSql.QSqlTableModel provides no direct support for foreign keys. Use the PySide2.QtSql.QSqlRelationalTableModel and PySide2.QtSql.QSqlRelationalDelegate if you want to resolve foreign keys.

The default edit strategy is OnRowChange .

PySide2.QtSql.QSqlTableModel.EditStrategy¶

This enum type describes which strategy to choose when editing values in the database.

Constant Description
QSqlTableModel.OnFieldChange All changes to the model will be applied immediately to the database.
QSqlTableModel.OnRowChange Changes to a row will be applied when the user selects a different row.
QSqlTableModel.OnManualSubmit All changes will be cached in the model until either PySide2.QtSql.QSqlTableModel.submitAll() or PySide2.QtSql.QSqlTableModel.revertAll() is called.

Note: To prevent inserting only partly initialized rows into the database, OnFieldChange will behave like OnRowChange for newly inserted rows.

Returns true if the row was deleted; otherwise returns false .

PySide2.QtSql.QSqlTableModel.insertRecord(row, record)¶
Parameters:
  • row – PySide2.QtCore.int
  • record – PySide2.QtSql.QSqlRecord
Return type:

PySide2.QtCore.bool

Inserts the record at position row . If row is negative, the record will be appended to the end. Calls PySide2.QtSql.QSqlTableModel.insertRows() and PySide2.QtSql.QSqlTableModel.setRecord() internally.

Returns true if the record could be inserted, otherwise false.

Changes are submitted immediately for OnFieldChange and OnRowChange . Failure does not leave a new row in the model.

PySide2.QtSql.QSqlTableModel.select()¶
Return type:PySide2.QtCore.bool

Populates the model with data from the table that was set via PySide2.QtSql.QSqlTableModel.setTable() , using the specified filter and sort condition, and returns true if successful; otherwise returns false .

Note

Calling will revert any unsubmitted changes and remove any inserted columns.

PySide2.QtSql.QSqlTableModel.setFilter(filter)¶
Parameters:filter – unicode

Sets the current filter to filter .

The filter is a SQL WHERE clause without the keyword WHERE (for example, name='Josephine') .

If the model is already populated with data from a database, the model re-selects it with the new filter. Otherwise, the filter will be applied the next time PySide2.QtSql.QSqlTableModel.select() is called.

PySide2.QtSql.QSqlTableModel.setRecord(row, record)¶
Parameters:
  • row – PySide2.QtCore.int
  • record – PySide2.QtSql.QSqlRecord
Return type:

PySide2.QtCore.bool

Applies values to the row in the model. The source and target fields are mapped by field name, not by position in the record.

Note that the generated flags in values are preserved to determine whether the corresponding fields are used when changes are submitted to the database. By default, it is set to true for all fields in a PySide2.QtSql.QSqlRecord . You must set the flag to false using PySide2.QtSql.QSqlRecord.setGenerated() (false) for any value in values , to save changes back to the database.

For edit strategies OnFieldChange and OnRowChange , a row may receive a change only if no other row has a cached change. Changes are submitted immediately. Submitted changes are not reverted upon failure.

Returns true if all the values could be set; otherwise returns false.

PySide2.QtSql.QSqlTableModel.submitAll()¶
Return type:PySide2.QtCore.bool

Submits all pending changes and returns true on success. Returns false on error, detailed error information can be obtained with PySide2.QtSql.QSqlQueryModel.lastError() .

In OnManualSubmit , on success the model will be repopulated. Any views presenting it will lose their selections.

Note: In OnManualSubmit mode, already submitted changes won’t be cleared from the cache when fails. This allows transactions to be rolled back and resubmitted without losing data.

PySide2.QtSql.QSqlTableModel.updateRowInTable(row, values)¶
Parameters:
  • row – PySide2.QtCore.int
  • values – PySide2.QtSql.QSqlRecord
Return type:

PySide2.QtCore.bool

Updates the given row in the currently active database table with the specified values . Returns true if successful; otherwise returns false .

This is a low-level method that operates directly on the database and should not be called directly. Use PySide2.QtSql.QSqlTableModel.setData() to update values. The model will decide depending on its edit strategy when to modify the database.

Note that only values that have the generated-flag set are updated. The generated-flag can be set with QSqlRecord.setGenerated() and tested with QSqlRecord.isGenerated() .
